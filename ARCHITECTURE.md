## Key Components

1. **Frontend (Next.js)**: User registration, status check, admin dashboard
2. **Django REST API**: Business logic, data validation, API endpoints
3. **MySQL Database**: Persistent data storage
4. **Celery Beat**: Schedules periodic reminder processing (every 15 min)
5. **Celery Worker**: Processes reminder sending tasks
6. **Redis**: Message broker for Celery task queue
7. **SMTP**: Email delivery via Gmail/other providers
8. **WhatsApp Business API**: Official WhatsApp message sending

## Automatic Processes

- **Every 15 minutes**: Celery Beat â†’ Check pending reminders â†’ Send if due
- **Daily at 2 AM**: Cleanup old sent/failed reminders (90+ days)
- **On Assignment**: Auto-create reminder with calculated scheduled_datetime
- **On Unlock**: Auto-cancel old reminder, log reason

## Safety Features

- Locked duties prevent accidental changes
- Emergency unlock requires mandatory reason
- UnlockLog provides complete audit trail
- Reminder status tracking prevents duplicates
- Retry logic handles network failures
- Timezone-aware datetimes prevent confusion
